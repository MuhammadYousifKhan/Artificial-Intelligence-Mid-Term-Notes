Week 1 — Introduction to AI + Intelligent Agents

Core theme: AI focuses on designing rational intelligent agents that perceive environments and act to maximize performance.

Important AI Definitions (from lecture)

  • • “Intelligence: ability to learn and solve problems” (Webster perspective).
  • • “AI is intelligence exhibited by machines/software” (Wikipedia style).
  • • “Science and engineering of making intelligent machines” (McCarthy).
  • • Russell & Norvig direction: design intelligent agents that perceive and act for success.

Four Classical AI Approaches

  • • Thinking humanly (cognitive modeling).
  • • Acting humanly (Turing test perspective).
  • • Thinking rationally (laws of thought, logic-heavy).
  • • Acting rationally (choose actions maximizing expected performance).

Applications Mentioned

Speech recognition, virtual assistants, handwriting recognition, machine translation, robotics, recommendation systems, search engines, email filtering, face detection/recognition, cancer imaging, game playing (Deep Blue/Watson/AlphaGo), autonomous driving, scheduling, fraud detection, NLP, medical diagnosis, route finding, protein design, document summarization.

Foundations of AI

  • • Philosophy (reasoning, rationality, mind).
  • • Mathematics (logic, probability, computation).
  • • Economics (decision theory, game theory, MDPs).
  • • Neuroscience + Psychology (thinking and behavior models).
  • • Computer engineering (hardware/system capability).
  • • Control theory/cybernetics and linguistics (feedback + language).

History Snapshot

  • • 1940–1950: Gestation (McCulloch-Pitts, Turing foundations).
  • • 1950–1970: Early enthusiasm (Dartmouth 1956, symbolic AI boom).
  • • 1970–1990: Knowledge-based AI + expert systems + AI winter.
  • • 1990–present: Scientific AI revival, probabilistic models, data-driven AI, deep learning.

Intelligent Agent Basics

  • • Agent = Architecture + Program.
  • • Sense → Think → Act loop.
  • • Agent function maps percept sequence to action.
  • • Rational agent picks action maximizing expected performance measure.
  • • Rationality depends on task measure, prior knowledge, available actions, and percept history.

PEAS (Task Environment Spec)

  • • P: Performance measure
  • • E: Environment
  • • A: Actuators
  • • S: Sensors

Self-driving car example: Performance (safety/time/legal/comfort), Environment (roads, cars, signs), Actuators (steering, brake, accelerator), Sensors (camera, GPS, sonar, odometer, etc.).

Week 2 — Search Agents and Problem Formulation

Core theme: convert a real problem into a formal search problem and then choose a strategy to explore it.

Problem Solving as Search (2-stage)

  1. Search phase (offline/mental exploration of possible actions).
  2. Execution phase (apply found solution).

Problem Formulation Components

  • • Initial state
  • • State space (all reachable states)
  • • Action space / Actions(s)
  • • Transition model Result(s, a)
  • • Goal test
  • • Path cost function

Canonical Examples

  • • 8-Queens (arrangements with non-attacking queens).
  • • 8-Puzzle (tile configurations and legal moves).
  • • Route finding (city-to-city on weighted graph).
  • • Traveling Salesperson (visit all cities shortest tour).
  • • VLSI layout and automatic assembly sequencing.
  • • Robot navigation in 2D/3D continuous spaces.
  • • Protein design by sequence search.

State Space vs Search Space

  • • State space = real problem configurations.
  • • Search space = abstract tree/graph explored by algorithm.
  • • Node stores parent, depth, state, and path cost.
  • • Expand(node): generates child nodes from available actions.

Search regions: Explored (closed), Frontier (open/fringe), Unexplored.

How Search Strategies Are Evaluated

Completeness: does it always find a solution if one exists?
Optimality: does it always return least-cost solution?
Time complexity: number of generated/expanded nodes.
Space complexity: max nodes held in memory.

Complexity symbols used in class: b (branching factor), d (depth of shallowest solution), m (max depth).

Week 3 — Uninformed Search (Blind Search)

Core theme: no heuristic/domain estimate is used. Expansion order defines behavior.

Uninformed Algorithms Cheat Matrix

Algorithm Node Priority Data Structure Complete? Optimal? Time Space
BFS Shallowest depth first FIFO Queue Yes (finite b) Yes if step costs equal O(b^d) O(b^d)
DFS Deepest node first LIFO Stack No in infinite-depth loops No O(b^m) O(bm)
DLS DFS up to limit l Stack + cutoff If l ≥ d No O(b^l) O(bl)
IDS Repeated DLS with growing limits Stack per iteration Yes Yes (unit costs) O(b^d) O(bd)
UCS Least path cost g(n) Priority Queue (min-heap) Yes (finite optimal cost) Yes O(b^(C*/ε)) O(b^(C*/ε))

Important Lecture Notes

  • • BFS is complete but memory explodes quickly with depth.
  • • DFS has tiny memory footprint but may miss shallow optimal paths.
  • • IDS combines BFS completeness + DFS low memory for unit-cost graphs.
  • • UCS handles weighted edges and can beat BFS in cost quality.
  • • UCS explores by increasing path cost, not level depth.

Map Example (Start: Las Vegas, Goal: Calgary)

Week 4 — Informed Search (Heuristic Search)

Core theme: use domain knowledge (heuristics) to guide expansion toward goals faster.

Heuristic Function Essentials

  • • h(n): estimated cost from node n to nearest goal.
  • • Heuristic can be imperfect; it only needs to be informative.
  • • Better-quality heuristics reduce search effort significantly.
  • • Example used in lecture map: straight-line distance to target city.

Greedy vs A*

  • • Greedy Best-First: f(n)=h(n) (chases estimated closeness only).
  • • A*: f(n)=g(n)+h(n) (balances traveled cost + estimated remainder).
  • • Greedy can be fast but not guaranteed optimal.
  • • A* is complete and optimal with admissible heuristic.

Admissible Heuristics

A heuristic is admissible if it never overestimates true remaining cost:

h(n) ≤ h*(n)

Lecture emphasizes that if h is admissible, A* tree search is optimal. Intuition: A* will always prioritize nodes that could still lead to a cheaper optimal goal than any known suboptimal goal node.

8-Puzzle Heuristics from Slides

  • • h1: number of misplaced tiles.
  • • h2: total Manhattan distance.
  • • Manhattan estimate shown as non-overestimating for the example state.

Lecture Recap Pairings

  • • BFS (layers by depth) ↔ IDS (subtrees by depth limit).
  • • UCS uses increasing path cost.
  • • Greedy uses increasing heuristic value.
  • • A* uses increasing g+h.
  • • IDA* does iterative deepening over g+h thresholds.

Week 4b — Local Search Deep Dive

Core theme: when path does not matter, optimize states directly using iterative improvement.

When Local Search Is Preferred

  • • Path to goal is irrelevant; only final state quality matters.
  • • Systematic tree search is too expensive or impossible.
  • • Very large/continuous optimization landscapes.
  • • Memory constraints are tight.

Lecture examples: integrated circuit design, telecom optimization, puzzle final configurations (e.g., 8-queens), etc.

Hill Climbing

  • • Greedy local search: keep one current state.
  • • Move to a better neighbor (uphill for maximization).
  • • Stops at local max / plateau / no improving move.
  • • Very low memory, often fast to get good-enough answer.

Common issues: local maxima, plateaus, ridges/flat regions.

Hill Climbing Variants

  • • Sideways moves: allow equal-value moves to escape plateaus.
  • • Random-restart: run hill climbing repeatedly from random starts.
  • • Stochastic hill climbing: pick random improving move.
  • • Local beam search: maintain k states in parallel.
  • • Stochastic beam search: sample successors probabilistically.

Simulated Annealing (concept)

  • • Inspired by physical annealing/cooling.
  • • Allows occasional worse moves early to escape local traps.
  • • As temperature drops, bad-move probability decreases.
  • • Good for rugged landscapes where greedy gets stuck.

P(accept worse move) ≈ exp(-ΔE / T)

Genetic Algorithms (GA)

  • • Population-based search (multiple individuals).
  • • Fitness function evaluates candidate quality.
  • • Parent selection favors fitter individuals.
  • • Crossover combines parent strings to create offspring.
  • • Mutation adds random diversity to avoid premature convergence.

Lecture 8-queens encoding: string of digits for queen row by column; fitness can be number of non-attacking pairs (max = 28).

Hill Climbing Skeleton

current = random_initial_state()
while True:
    neighbor = best_neighbor(current)
    if value(neighbor) <= value(current):
        return current
    current = neighbor

Genetic Algorithm Skeleton

population = init_random_population(k)
repeat until stop_condition:
    parents = select_by_fitness(population)
    children = crossover(parents)
    children = mutate(children, prob)
    population = survivor_selection(population, children)
return best_individual(population)

Revision Pack (Quick Exam Prep)

High-Value One-Liners

  • • Rational agent chooses actions that maximize expected performance measure.
  • • PEAS is the full task-environment specification for agent design.
  • • Search problem = initial state + actions + transition + goal test + path cost.
  • • BFS is complete/optimal (unit costs) but memory-expensive.
  • • DFS uses very low memory but can fail in infinite-depth spaces.
  • • UCS expands cheapest path cost first; optimal with positive step costs.
  • • Greedy uses only heuristic and can be fast but suboptimal.
  • • A* with admissible heuristic is complete and optimal.
  • • Local search ignores path details and optimizes final state quality.
  • • Genetic algorithms use population + fitness + crossover + mutation.
Frequently Asked Viva Questions
  • 1) Why is AI usually taught via rational agents? → It gives an actionable, measurable design objective.
  • 2) Difference between state space and search space? → Real configurations vs explored tree/graph abstraction.
  • 3) Why can BFS be impractical? → Exponential time and exponential memory.
  • 4) Why can DFS be useful? → Tiny memory footprint and fast deep discovery in dense solutions.
  • 5) Why is UCS preferred over BFS on weighted maps? → Finds cheapest path, not shortest depth.
  • 6) What is an admissible heuristic? → Never overestimates true remaining cost.
  • 7) Why is A* optimal with admissible h? → It never commits to a suboptimal goal too early.
  • 8) Why local search for 8-queens optimization? → Final arrangement matters more than path.
  • 9) How does random-restart help hill climbing? → Escapes local maxima by new starting points.
  • 10) Role of mutation in GA? → Maintains diversity and prevents premature convergence.
Mini Formula Sheet

  • • Greedy expands min h(n)
